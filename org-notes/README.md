# Org-Notes: Приложение для ведения заметок

## Описание
Org-Notes — это кроссплатформенное мобильное приложение с графическим пользовательским интерфейсом, разработанное для удобного создания, редактирования, удаления и поиска заметок в рамках организационной деятельности (совещания, собрания, конференции и т.д.). Приложение поддерживает два режима хранения данных: в локальной базе данных SQLite или в виде файлов на устройстве.

## Основные Функциональные Возможности

### 1. Создание Заметки
*   **Действие:** Нажмите кнопку `+` (плюс) на главном экране.
*   **Поля:** Укажите заголовок, содержание заметки. Дата и время создания заполняются автоматически.
*   **Сохранение:** Нажмите иконку сохранения (дискета) или кнопку "Создать Заметку" для сохранения.

### 2. Редактирование Заметки
*   **Действие:** На главном экране нажмите на иконку карандаша рядом с заметкой или кликните по заметке для перехода на экран редактирования.
*   **Изменение:** Отредактируйте заголовок или содержание.
*   **Сохранение:** Нажмите иконку сохранения (дискета) или кнопку "Сохранить Изменения".

### 3. Удаление Заметки
*   **Действие:** На главном экране нажмите на иконку корзины рядом с заметкой.
*   **Подтверждение:** Подтвердите удаление во всплывающем окне.

### 4. Поиск Заметок
*   **Действие:** Используйте строку поиска в верхней части главного экрана.
*   **Параметры:** Поиск осуществляется по заголовку, содержанию, дате или времени.

### 5. Выбор Способа Хранения Данных
*   **Действие:** На главном экране нажмите на иконку настроек (шестеренка) в правом верхнем углу.
*   **Опции:** Выберите между "SQLite (локальная база данных)" и "Файловая система (файлы JSON)".
*   **Важно:** Переключение между типами хранилища **не переносит существующие заметки**. Заметки, созданные в одном типе хранилища, будут доступны только при выбранном этом типе.

## Установка и Запуск Приложения (для разработчиков)

Это приложение создано с использованием Expo (React Native).

1.  **Клонируйте репозиторий:** 
    ```bash
    git clone [ссылка на ваш репозиторий]
    cd org-notes
    ```
    (предполагается, что вы находитесь в папке `F:\JS_proj\notes`)
    ```bash
    cd org-notes
    ```

2.  **Установите зависимости:** 
    ```bash
    npm install
    ```

3.  **Запустите приложение:** 
    ```bash
    npm start
    ```
    Это запустит Metro Bundler и откроет страницу Expo Dev Tools в вашем браузере.

## Тестирование Приложения (для научного руководителя)

Для тестирования приложения без необходимости установки всей среды разработки:

1.  **Установите приложение Expo Go:**
    *   На ваш смартфон (iOS или Android) установите бесплатное приложение "Expo Go" из App Store или Google Play Store.

2.  **Запустите приложение на компьютере:**
    *   Перейдите в директорию `org-notes` (например, `cd F:\JS_proj\notes\org-notes`).
    *   Выполните команду `npm start`.
    *   В вашем браузере откроется страница Expo Dev Tools.

3.  **Сканируйте QR-код:**
    *   На странице Expo Dev Tools вы увидите QR-код.
    *   Откройте приложение "Expo Go" на вашем смартфоне и используйте его для сканирования этого QR-кода.
    *   Приложение Org-Notes загрузится на вашем смартфоне.

**Ссылка на приложение:**
Поскольку приложение запущено локально на вашей машине, прямая ссылка невозможна. Тестирование осуществляется путем сканирования QR-кода через Expo Go, как описано выше.

### Как проверить функциональные возможности:

1.  **Создайте несколько заметок:** Попробуйте создать 2-3 заметки с разными заголовками и содержанием.
2.  **Отредактируйте заметку:** Измените текст или заголовок одной из созданных заметок.
3.  **Удалите заметку:** Удалите одну из заметок.
4.  **Протестируйте поиск:**
    *   Введите часть заголовка или содержания в строку поиска.
    *   Попробуйте найти заметку по точной дате (например, "2025-11-29") или времени (например, "15:00").
5.  **Проверьте переключение хранилища:**
    *   Создайте 2-3 заметки при выбранном типе хранилища "SQLite".
    *   Перейдите в "Настройки" и переключитесь на "Файловая система".
    *   Убедитесь, что заметки, созданные в SQLite, **не видны**.
    *   Создайте 2-3 новые заметки при выбранном "Файловая система".
    *   Переключитесь обратно на "SQLite" и убедитесь, что заметки из файловой системы **не видны**, а предыдущие заметки из SQLite **снова доступны**.

## Архитектура и Хранение Данных

Приложение использует гибкую архитектуру, позволяющую переключаться между двумя основными способами хранения:

1.  **SQLiteStorageService:** Использует библиотеку `expo-sqlite` для работы с локальной базой данных SQLite. Данные хранятся в таблице `notes`.
2.  **FileSystemStorageService:** Использует `expo-file-system` для сохранения каждой заметки в виде отдельного JSON-файла в приватной директории приложения на устройстве. Имя файла соответствует ID заметки (`{id}.json`).

Выбор между этими сервисами абстрагирован через `StorageManager`, который также сохраняет предпочтительный тип хранилища с помощью `AsyncStorage`. Это демонстрирует модульный подход к управлению данными, позволяющий легко добавлять новые типы хранилищ в будущем.
